Порівняння роботи алгоритмів DFS та BFS
Для аналізу я змоделював граф київського метрополітену. Завдання: Знайти маршрут від ст. "Лісова" до ст. "Теремки".

1. Алгоритм BFS (Пошук у ширину)
Результат: Знайшов найкоротший шлях (мінімальна кількість пересадок).

Маршрут: від станції "Лісова" -> "Хрещатик" -> (пересадка) -> "Майдан Незалежності" -> "Теремки".

Чому так: BFS працює як хвиля на воді. Він одночасно перевіряє всі найближчі станції. Тому він гарантовано знаходить шлях з найменшою кількістю ребер (зупинок/пересадок). У моєму випадку він відразу побачив пересадку на Хрещатику і не пішов перевіряти довші варіанти.

2. Алгоритм DFS (Пошук у глибину)
Результат: Знайшов довший, неоптимальний шлях.

Маршрут: від станції "Лісова" -> "Театральна" -> (пересадка) -> "Золоті Ворота" -> "Палац Спорту" -> (пересадка) -> "Площа Укр. Героїв" -> "Теремки" (приклад).

Чому так: DFS працює як людина в лабіринті, яка тримається однієї стіни. Він йде по одній гілці до упору. Дійшовши до станції "Театральна", алгоритм "пірнув" у пересадку на зелену гілку і пішов досліджувати її, ігноруючи той факт, що можна було проїхати одну зупинку до Хрещатика і пересісти зручніше. DFS не шукає найкоротший шлях, він просто шукає будь-який шлях.

Висновок:
BFS ідеальний для знаходження найкоротших маршрутів у навігаторах та мапах метро.

DFS краще підходить для перевірки всіх можливих варіантів або пошуку циклів, але не гарантує оптимальності маршруту.
